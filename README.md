LEVANTMENTO DE REQUISITOS:



 RF1-LOGIN:
O sistema deve permitis login dos alunos utilizando RA e senha
Já o professor deve fazer login utilizando seu email institucional e senha

RF2-RELATÓRIO DE DEFEITOS/BATERIA FRACA:

O SISTEMA DEVERÁ PERMITIR QUE O ALUNO ESCANEIE UM QR CODE COLADO NAS MAQUINAS QUE O LEVE ATÉ UM FORMULÁRIO ONDE PODERÁ RELATAR UM PROBLEMA, OU ENTÃO QUE O USUÁRIO NAVEGUE PELO SISTEMA E RELATE O PROBLEMA


RF3-ORGANIZAÇÃO:
O SISTEMA DEVERÁ CONTAR COM 2 CARRINHOS QUE ABRIGARÃO AS MÁQUINAS(ESTES SÃO CARRINHOS 4 E 5)


RF4-ASSOCIAÇÃO DE PROFESSORES:
O SISTEMA DEVERÁ RECONHECER A QUAIS PROFESSORES OS CARRINHOS SÃO ASSOCIADOS
(NO NOSSO CASO CARRINHO 4 = MARCOS E FÁBIO, CARRINHO 5 = BETO E PEDRO)


RF5-VISUALIZAÇÃO DE MÁQUINAS:
O SISTEMA DEVERÁ PERMITIR QUE O PROFESSOR TENHA UM DASHBOARD EXIBINDO VISUALMENTE AS MÁQUINAS, PERMITINDO VISUALIZAR O STATUS


RF6-CONTROLE DE STATUS:
O SISTEMA DEVERÁ PERMITIR QUE OS PROFESSORES ALTEREM O STATUS DAS MÁQUINAS, PODENDO MUDAR AO RECEBER A NOTIFICAÇÃO DO DEFEITO DE UMA MÁQUINA OU ALTERAR QUANDO UMA MÁQUINA DANIFICADA FOR CONSERTADA


RF7-HISTÓRICO DE MANUTENÇÃO

O SISTEMA DEVERÁ PERMITIR QUE O PROFESSOR ACESSA UM HISTÓRICO QUE RALATA DEFEITOS


RF8-GERENCIAMENTO DE RELATÓRIOS
Os professores deverão conseguir visualizar e atualizar relatórios de defeitos


REQUISITOS NÃO FUNCIONAIS



RNF1- RESPONSIVIDADE:
O TAMANHO DA TELA DEVE SER ADAPTÁVEL AO TAMANHO DO DISPOSITIVO

RNF2- INTERFACE INTUITIVA:
O SISTEMA DEVERÁ SER FACILMENTE UTILIZAD COM UMA INTERFACE SIMPLES

RNF3-SEGURANÇA:
O SISTEMA DEVERÁ CONTAR COM AUTENTICAÇÕES E TER CONTROLE DE PERMISSÕES, NÃO PERMITINDO OS ALUNOS ACESSAREM ÁREAS RESTRITAS AOS PROFESSORES
